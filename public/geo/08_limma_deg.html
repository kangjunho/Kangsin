Step 08

limma 차등발현 분석

분석 질문을 design과 contrast로 표현하고, Disease와 Control의 probe-level 발현 차이를 검정합니다.

차등발현 분석의 질문

차등발현 분석은 각 probe에 대해 관찰된 Disease와 Control의 평균 발현차이가 표본 간 생물학적 변동과 측정오차를 고려한 뒤에도 충분히 큰지 검정합니다.

H0: Disease와 Control의 평균 발현이 같다.
H1: Disease와 Control의 평균 발현이 다르다.
limma의 핵심: 각 probe에 선형모형을 적합한 뒤, probe 전체의 정보를 이용하는 empirical Bayes 방법으로 분산 추정을 안정화합니다. 표본 수가 많지 않은 microarray 연구에 특히 유용합니다.

4. Contrast 방향

contrast_matrix <- makeContrasts(
  Disease_vs_Control = Disease - Control,
  levels = design
)

contrast_matrix
logFC > 0  → Disease에서 발현 증가
logFC < 0  → Disease에서 발현 감소
Control - Disease로 작성하면 부호가 반대가 됩니다. 분석 전에 contrast 이름, 수식과 행렬을 출력하고 연구 질문의 방향과 일치하는지 확인합니다.

5. limma 모형 적합

fit_base <- lmFit(
  expr_probe,
  design
)
fit_contrast <- contrasts.fit(
  fit_base,
  contrast_matrix
)
fit_ebayes <- eBayes(
  fit_contrast
)
함수역할
lmFit()각 probe에 design matrix로 표현된 선형모형을 적합합니다.
contrasts.fit()추정된 그룹 계수를 Disease − Control 비교로 변환합니다.
eBayes()probe 간 정보를 이용해 분산을 안정화하고 moderated t, P-value와 B 값을 계산합니다.

eBayes(robust=TRUE) 또는 trend=TRUE는 항상 더 좋은 설정이 아닙니다. 분산 이상치와 평균–분산 관계를 확인하고 분석 계획에 따라 선택합니다.

8. DEG 기준 설정

fdr_cutoff <- 0.05
lfc_cutoff <- 1

deg_annotated$status <- "Not_significant"
deg_annotated$status[
  deg_annotated$adj.P.Val < fdr_cutoff &
    deg_annotated$logFC >= lfc_cutoff
] <- "Up"
deg_annotated$status[
  deg_annotated$adj.P.Val < fdr_cutoff &
    deg_annotated$logFC <= -lfc_cutoff
] <- "Down"

table(deg_annotated$status)

|logFC| ≥ 1은 2배 이상의 발현차이를 의미하지만 모든 연구에 적용되는 절대 기준은 아닙니다. 조직, 플랫폼, 표본 수와 연구 목적을 고려해 분석 전에 정합니다.

FDR 유의 DEG가 없을 수도 있습니다. 이때 결과가 없다는 이유로 사후에 기준을 raw P-value로 변경하여 “유의한 DEG”라고 보고하면 다중검정 오류가 커집니다. 탐색적 nominal P-value 결과를 제시한다면 FDR을 통과하지 못했다는 사실을 명확히 구분해야 합니다.

9. 단순 cutoff와 treat()의 차이

일반적인 방법은 FDR을 먼저 계산한 뒤 결과표에서 |logFC| cutoff를 추가합니다. treat()는 발현차이가 지정한 최소 효과크기보다 큰지를 통계모형 안에서 직접 검정합니다.

fit_treat <- treat(
  fit_contrast,
  lfc = 1
)

treat_result <- topTreat(
  fit_treat,
  coef = "Disease_vs_Control",
  number = Inf,
  sort.by = "p"
)
방법질문
eBayes() + 결과표 cutoff발현차이가 0과 다른가? 그중 |logFC| 기준도 만족하는가?
treat(lfc=1)발현차이가 통계적으로 |logFC| 1보다 큰가?

두 결과는 동일하지 않을 수 있습니다. 논문에서는 사용한 방법과 기준을 정확히 명시합니다.

10. decideTests()는 무엇이 다른가?

decision <- decideTests(
  fit_ebayes,
  adjust.method = "BH",
  p.value = 0.05,
  lfc = 1
)

summary(decision)

decideTests()는 각 contrast의 결과를 −1, 0, 1로 요약하므로 여러 contrast를 비교하거나 Venn diagram을 만들 때 편리합니다. 직접 만든 status 열은 각 probe의 annotation과 함께 CSV로 저장하기 편리합니다.

12. Full-rank 오류와 confounding

qr(design_cov)$rank
ncol(design_cov)

if (qr(design_cov)$rank < ncol(design_cov)) {
  stop("Design matrix가 full rank가 아닙니다.")
}

Coefficients not estimable 또는 design matrix not of full rank 오류는 일부 변수가 다른 변수와 완전히 중복되어 독립적인 효과를 추정할 수 없을 때 발생합니다.

예시문제
모든 Disease는 batch 1, 모든 Control은 batch 2Disease 효과와 batch 효과를 분리할 수 없음
sex 열과 male_indicator 열을 동시에 포함동일한 정보를 중복 표현
표본 수보다 지나치게 많은 공변량잔여 자유도 부족
한 수준만 남은 factor해당 변수의 효과를 비교할 수 없음

13. Matched와 paired design 구분

Matched case-control은 연령·성별 등이 비슷한 서로 다른 사람을 선택한 설계일 수 있습니다. 반면 paired/repeated design은 같은 사람에게서 치료 전·후 또는 여러 시점의 sample을 측정한 설계입니다.

설계동일 subject?분석
나이·성별을 맞춘 Disease와 Control아니오독립표본 + 필요한 공변량
동일 환자의 치료 전·후예patient blocking 또는 paired model
동일 환자의 여러 조직·시점예반복측정 구조 반영
 
block <- factor(
  sample_info$subject_id
)

corfit <- duplicateCorrelation(
  expr_probe,
  design,
  block = block
)

fit_pair <- lmFit(
  expr_probe,
  design,
  block = block,
  correlation = corfit$consensus
)
제목에 “matched”가 있다는 이유만으로 paired analysis를 적용하지 않습니다. 동일 subject ID가 여러 GSM에 실제로 반복되는지 확인해야 합니다.

종합 실습

  1. Control과 Disease의 sample 수를 확인하고 design matrix를 직접 설명합니다.
  2. Disease-Control contrast에서 양수·음수 logFC의 의미를 작성합니다.
  3. 전체 probe 결과에 gene symbol을 붙이고 annotation 성공률을 계산합니다.
  4. FDR 0.05와 |logFC| ≥ 1에서 Up·Down DEG 수를 기록합니다.
  5. FDR 0.01·0.05·0.10과 logFC 0·0.5·1 조합에 따른 DEG 수를 비교합니다.
  6. eBayes() 결과표 cutoff와 treat(lfc=1) 결과가 다른 이유를 설명합니다.
  7. 사용 가능한 임상정보 중 design에 포함할 공변량과 그 이유를 제시합니다.
  8. 현재 자료가 독립표본인지 실제 paired/repeated design인지 근거를 작성합니다.
