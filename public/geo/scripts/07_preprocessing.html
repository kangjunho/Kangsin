GEO Microarray Analysis 과정 목차
Step 07

전처리, QC와 probe annotation

발현값 분포와 sample 관계를 점검하고, 분석 가능한 probe·gene expression matrix를 만듭니다.

전처리의 목적

전처리는 단순히 함수를 실행하는 과정이 아니라, 현재 발현값이 어떤 상태인지 확인하고 차등발현 분석에 사용할 수 있는 형태로 만드는 과정입니다.

값의 척도→ 샘플 QC→ probe annotation→ 분석 행렬
Series Matrix는 보통 processed data입니다. 이미 log2 변환과 normalization이 적용되었을 수 있으므로 raw data 전처리 과정을 무조건 다시 적용하면 안 됩니다.

2. Raw data와 processed data 구분

구분예일반적인 처리
Raw microarray dataAffymetrix CEL, Illumina IDAT, Agilent TXTbackground correction, normalization, summarization과 QC를 플랫폼별로 수행
Processed Series MatrixGEO2R와 getGEO(GSEMatrix=TRUE)의 발현값제출자가 수행한 처리방법을 확인하고 필요한 점검만 수행
이번 과정: processed Series Matrix를 사용합니다. Raw CEL·IDAT 전처리는 플랫폼마다 다르므로 별도의 심화과정으로 분리합니다.

3. log2 변환 여부 판단

qx <- quantile(
  expr_raw,
  c(0, 0.25, 0.5, 0.75, 0.99, 1),
  na.rm = TRUE
)
qx

log2_needed <- (qx[5] > 100) ||
  ((qx[6] - qx[1] > 50) && qx[2] > 0)

expr_work <- expr_raw
if (log2_needed) {
  expr_work[expr_work <= 0] <- NA_real_
  expr_work <- log2(expr_work)
}
자동 판단식은 보조 기준입니다. 최종 결정은 GSE의 Data processing, 값의 범위, boxplot과 density plot을 함께 보고 내립니다. 이미 log2인 값을 다시 변환하면 발현차이와 분포가 왜곡됩니다.

4. 결측값과 비정상 값

sum(is.na(expr_work))
sum(is.infinite(expr_work))

missing_by_sample <- colSums(is.na(expr_work))
missing_by_probe <- rowSums(is.na(expr_work))

# 사용할 값이 하나도 없는 probe 제거
keep_nonempty <- rowSums(!is.na(expr_work)) > 0
expr_work <- expr_work[
  keep_nonempty,
  ,
  drop = FALSE
]

결측값이 일부 있다는 이유로 모든 probe 또는 sample을 자동 제거하지 않습니다. 특정 sample에 결측이 집중되는지, 플랫폼의 detection filtering 때문인지 확인합니다.

QC용 complete matrix와 DEG용 matrix를 구분합니다. PCA처럼 결측값을 허용하지 않는 분석에는 complete probe만 임시로 사용하고, DEG 입력에서 어떤 probe를 제거할지는 별도로 기록합니다.

7. PCA로 주요 변동 확인

probe_variance <- apply(
  expr_complete,
  1,
  var
)
probe_variance <- sort(
  probe_variance,
  decreasing = TRUE
)

top_n <- min(5000, length(probe_variance))
top_probe_ids <- names(probe_variance)[
  seq_len(top_n)
]

pca <- prcomp(
  t(expr_complete[top_probe_ids, ]),
  center = TRUE,
  scale. = FALSE
)

PCA는 가장 큰 발현 변동을 요약합니다. Disease와 Control 분리는 분석의 성공 여부를 직접 증명하지 않으며, 분리되지 않는다고 DEG가 존재하지 않는 것도 아닙니다.

PCA에서 보이는 패턴확인사항
그룹별 분리질환 효과인지 batch·조직·성별 차이인지 metadata와 대조
한 sample이 멀리 떨어짐품질, protocol, 진단, 시점과 입력 오류 확인
batch별 분리batch가 그룹과 독립적인지, design에 포함 가능한지 확인
연속적인 분포연령, severity 또는 cell composition 같은 연속형 요인 확인

8. 추가 정규화가 필요한가?

# 기본값: processed Series Matrix에 재정규화하지 않음
force_normalization <- FALSE

if (force_normalization) {
  expr_work <- limma::normalizeBetweenArrays(
    expr_work,
    method = "quantile"
  )
}
분포가 조금 다르다는 이유만으로 quantile normalization을 적용하지 않습니다. 원 자료의 처리방법, platform 특성과 QC 결과를 확인한 뒤 결정하고, 적용했다면 분석방법에 명시합니다.

서로 다른 platform의 Series Matrix를 단순히 합친 뒤 정규화하는 것은 권장되지 않습니다. probe 정의와 측정특성이 다르므로 gene-level 변환, 연구별 분석과 meta-analysis 등의 별도 전략이 필요합니다.

9. Batch effect는 어떻게 다루나?

Batch effect는 실험 날짜, chip, 연구기관 또는 처리방법처럼 관심 조건 이외의 기술적 변동입니다.

상황권장 접근
batch가 metadata에 있고 그룹과 겹치지 않음limma design에 batch 공변량 포함
시각화에서 batch를 제거하고 싶음removeBatchEffect() 결과를 PCA·heatmap용으로 사용
batch와 Disease가 완전히 일치두 효과를 통계적으로 분리하기 어려우므로 데이터셋 설계 재검토
 
# 시각화용 예시이며 원본 DEG 입력을 대체하지 않음
expr_visual <- limma::removeBatchEffect(
  expr_work,
  batch = analysis_meta$batch,
  design = model.matrix(
    ~ analysis_meta$group
  )
)
DEG 분석에서는: 가능한 경우 발현값에서 batch를 먼저 제거하기보다 Step 08의 design matrix에 batch를 포함하여 모형 안에서 보정합니다.

10. Probe annotation 열 찾기

colnames(feature_info)

symbol_candidates <- c(
  "Gene.symbol", "Gene.Symbol", "Gene Symbol",
  "GENE_SYMBOL", "SYMBOL", "Symbol"
)
symbol_col <- intersect(
  symbol_candidates,
  colnames(feature_info)
)

if (length(symbol_col) == 0) {
  symbol_col <- grep(
    "gene.*symbol|symbol",
    colnames(feature_info),
    value = TRUE,
    ignore.case = TRUE
  )
}

symbol_col <- symbol_col[1]

플랫폼마다 gene symbol 열 이름이 다르므로 실제 열을 확인해야 합니다. 자동으로 선택된 열에 정말 gene symbol이 들어 있는지도 head(feature_info[[symbol_col]])로 확인합니다.

org.Hs.eg.db가 모든 microarray probe를 직접 변환해 주지는 않습니다. GPL annotation 또는 GPL570의 경우 hgu133plus2.db처럼 해당 platform 전용 annotation package를 사용하는 방법을 검토합니다.

11. Probe와 gene의 관계

Microarray에서는 하나의 gene을 여러 probe가 측정할 수 있고, 하나의 probe가 여러 gene에 연결되는 경우도 있습니다.

상황문제처리 예
symbol이 비어 있음gene 수준 해석 불가gene matrix에서 제외하되 probe DEG 결과는 보존
GENE1 /// GENE2한 probe의 유전자 대응이 모호함입문 분석에서는 제외하거나 별도 검토
한 gene에 여러 probegene이 결과표에 중복됨평균, 중앙값 또는 최대 변이 probe 선택
probe 간 방향이 반대isoform, 품질 또는 annotation 문제 가능무조건 평균하지 말고 probe 위치와 결과 확인

mapped_variance <- apply(
  expr_mapped,
  1,
  var,
  na.rm = TRUE
)
ord <- order(
  mapped_variance,
  decreasing = TRUE
)
keep <- !duplicated(
  probe_map_valid$gene_symbol[ord]

gene_matrix_maxvar <- expr_mapped[
  ord[keep],
  ,
  drop = FALSE

권장 분리: Step 08의 limma DEG는 probe 수준 행렬로 수행하고 annotation을 붙이는 흐름을 기본으로 합니다. Gene-level 행렬은 heatmap, pathway score와 데이터 통합에 사용할 수 있습니다. Gene 수준에서 DEG를 수행한다면 병합 기준을 분석 전에 정하고 Methods에 기록해야 합니다.
