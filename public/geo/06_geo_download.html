GEO Microarray Analysis 과정 목차
Step 06

GEOquery로 데이터와 임상정보 가져오기

GSE의 발현행렬, sample metadata와 probe annotation을 분리하고 분석용 그룹 정보를 만듭니다.

이번 단계의 결과물

Step 05에서는 GEO2R 코드를 재현했습니다. 이번 단계에서는 GEO 자료를 R 객체로 가져온 뒤, 다음 세 종류의 정보를 서로 연결하여 저장합니다.

발현행렬+ 샘플·임상정보+ probe annotation
  • expression_matrix.csv: probe × sample 발현값
  • sample_metadata_full.csv: GEO가 제공한 전체 sample metadata
  • sample_metadata_analysis.csv: 분석에 사용할 GSM과 그룹 정보
  • characteristics_long.csv: characteristics를 항목·값으로 분리한 표
  • feature_annotation.csv: probe annotation
  • expression_set.rds: 원래 ExpressionSet 객체

1. GSE 다운로드

library(GEOquery)
library(Biobase)

options(timeout = 600)
options(download.file.method = "libcurl")

gse_id <- "GSE54568"
cache_dir <- "C:/GEO_cache"
dir.create(cache_dir, recursive = TRUE, showWarnings = FALSE)

gset_list <- getGEO(
  gse_id,
  GSEMatrix = TRUE,
  AnnotGPL = TRUE,
  destdir = cache_dir
)

getGEO()의 결과가 바로 하나의 발현행렬이 아니라 ExpressionSet을 담은 list인 이유는 하나의 GSE에 여러 GPL이 포함될 수 있기 때문입니다.

인수의미
GSEMatrix = TRUESeries Matrix를 ExpressionSet 형태로 불러옵니다.
AnnotGPL = TRUE가능한 경우 NCBI가 생성한 GPL annotation을 feature 정보에 추가합니다.
destdir다운로드 파일을 저장할 cache 폴더입니다.
Step 05의 다운로드 오류가 다시 발생한다면: 먼저 C:/GEO_cache 폴더와 네트워크를 확인하고, Series Matrix를 수동 다운로드하는 방법을 사용합니다.

분석하려는 GPL이 GPL570이라면 이름이나 annotation() 결과를 이용해 명시적으로 선택합니다.

target_gpl <- "GPL570"
gpl_ids <- vapply(gset_list, annotation, character(1))
idx <- which(gpl_ids == target_gpl)

if (length(idx) != 1) {
  stop("분석할 GPL을 하나만 선택하지 못했습니다.")
}

eset <- gset_list[[idx]]
annotation(eset)
dim(eset)
gset_list[[1]]을 무조건 사용하지 않습니다. list의 첫 번째 객체가 원하는 GPL이라는 보장이 없습니다. GSE 웹페이지에서 확인한 platform ID, sample 수와 R 객체가 일치하는지 확인하세요.

3. ExpressionSet 이해하기

ExpressionSet은 발현값과 sample·feature 정보를 연결해서 보관하는 Bioconductor의 객체입니다.

expr_matrix  <- exprs(eset)
sample_info  <- pData(eset)
feature_info <- fData(eset)

dim(expr_matrix)
dim(sample_info)
dim(feature_info)
함수행열내용
exprs(eset)probeGSM sampleprocessed expression value
pData(eset)GSM samplemetadata 항목title, source, characteristics, platform 등
fData(eset)probeannotation 항목probe ID, gene symbol, gene title 등
연결 기준: colnames(expr_matrix)와 rownames(sample_info)는 GSM accession으로 연결되고, rownames(expr_matrix)와 rownames(feature_info)는 probe ID로 연결됩니다.

4. 가장 먼저 수행할 일치성 검사

identical(
  colnames(expr_matrix),
  rownames(sample_info)
)

identical(
  rownames(expr_matrix),
  rownames(feature_info)
)

stopifnot(
  identical(colnames(expr_matrix), rownames(sample_info)),
  identical(rownames(expr_matrix), rownames(feature_info))
)

두 객체에 같은 GSM이 있어도 순서가 다르면 그룹정보가 잘못 연결될 수 있습니다. 순서가 다른 경우 metadata를 발현행렬 순서에 맞추어 재배열합니다.

sample_info <- sample_info[
  match(colnames(expr_matrix), rownames(sample_info)),
  ,
  drop = FALSE
]

stopifnot(
  identical(colnames(expr_matrix), rownames(sample_info))
)
단순히 sort()하지 마세요. 발현행렬과 metadata를 각각 정렬하면 우연히 맞을 수도 있지만, 일부 샘플이 누락된 경우 잘못 연결될 수 있습니다. 기준 객체의 ID에 match()하는 방법이 안전합니다.

6. characteristics를 임상정보 표로 분리하기

예를 들어 metadata에 다음과 같은 문자열이 저장될 수 있습니다.

subject id: 101
disease: MDD
age: 45
gender: female
tissue: brain

먼저 모든 characteristics 열을 GSM–항목–값의 긴 형식으로 변환합니다.

char_cols <- grep(
  "^characteristics_ch1",
  colnames(sample_info),
  value = TRUE
)

char_long <- do.call(
  rbind,
  lapply(seq_len(nrow(sample_info)), function(i) {
    values <- unlist(
      sample_info[i, char_cols, drop = FALSE],
      use.names = FALSE
    )
    values <- trimws(as.character(values))
    values <- values[!is.na(values) & nzchar(values)]

    data.frame(
      GSM = rownames(sample_info)[i],
      field = trimws(sub(":.*$", "", values)),
      value = trimws(sub("^[^:]*:[[:space:]]*", "", values)),
      raw = values,
      stringsAsFactors = FALSE
    )
  })
)

head(char_long)
sort(unique(char_long$field))
콜론이 없는 값도 확인하세요. 모든 characteristics가 반드시 항목: 값 형식은 아닙니다. raw 열을 보존하여 분리 결과가 원문과 일치하는지 확인합니다.

7. 분석용 Disease·Control 그룹 만들기

GSE54568 예시에서는 title에 MDD와 Control 정보가 들어 있으므로 다음과 같이 후보 그룹을 만들 수 있습니다.

analysis_meta <- data.frame(
  GSM = rownames(sample_info),
  title = sample_info$title,
  source = sample_info$source_name_ch1,
  stringsAsFactors = FALSE
)

analysis_meta$group <- NA_character_
analysis_meta$group[
  grepl("MDD", analysis_meta$title, ignore.case = TRUE)
] <- "Disease"
analysis_meta$group[
  grepl("control", analysis_meta$title, ignore.case = TRUE)
] <- "Control"

table(analysis_meta$group, useNA = "ifany")
analysis_meta[, c("GSM", "title", "group")]
이 코드는 GSE54568에 맞춘 예시입니다. 다른 데이터셋에 그대로 적용하지 말고 실제 metadata 값에 맞게 검색어와 조건을 수정하세요. grepl() 결과를 GSM별로 눈으로 확인한 후 그룹을 확정합니다.

그룹 순서를 명확히 지정하면 이후 design matrix에서 기준을 일관되게 관리할 수 있습니다.

analysis_meta$group <- factor(
  analysis_meta$group,
  levels = c("Control", "Disease")
)

stopifnot(!anyNA(analysis_meta$group))
table(analysis_meta$group)

factor의 첫 level을 Control로 둔 것은 metadata 관리 방식입니다. limma에서 최종 logFC 방향은 다음 단계에서 작성할 Disease-Control contrast가 결정합니다.

8. 반복측정과 동일 환자 확인

GSM 수와 환자 수는 같지 않을 수 있습니다. characteristics에 subject ID가 있다면 같은 대상자가 여러 시점에 반복되는지 확인합니다.

subject_rows <- char_long[
  tolower(char_long$field) %in% c(
    "subject id", "subject_id", "patient id", "patient_id"
  ),
  c("GSM", "value")
]
colnames(subject_rows)[2] <- "subject_id"

table(subject_rows$subject_id)
subject_rows[duplicated(subject_rows$subject_id) |
               duplicated(subject_rows$subject_id, fromLast = TRUE), ]
반복측정이 발견되면: 독립 2그룹 분석보다 patient blocking 또는 paired design이 필요할 수 있습니다. 동일 subject의 치료 전·후 샘플을 서로 독립된 환자로 분석하면 분산과 P-value가 잘못 계산될 수 있습니다.

9. 원본과 분석용 파일 저장

output_dir <- "data_processed"
dir.create(output_dir, recursive = TRUE, showWarnings = FALSE)

# R 객체 구조를 그대로 보존
saveRDS(
  eset,
  file.path(output_dir, "expression_set.rds")
)

# Excel에서 한글이 깨지지 않도록 UTF-8-BOM CSV 저장
expression_export <- data.frame(
  PROBE_ID = rownames(expr_matrix),
  expr_matrix,
  check.names = FALSE
)
write.csv(
  expression_export,
  file.path(output_dir, "expression_matrix.csv"),
  row.names = FALSE,
  fileEncoding = "UTF-8-BOM"
)
write.csv(
  cbind(GSM = rownames(sample_info), sample_info),
  file.path(output_dir, "sample_metadata_full.csv"),
  row.names = FALSE,
  fileEncoding = "UTF-8-BOM"
)
write.csv(
  analysis_meta,
  file.path(output_dir, "sample_metadata_analysis.csv"),
  row.names = FALSE,
  fileEncoding = "UTF-8-BOM"
)
write.csv(
  char_long,
  file.path(output_dir, "characteristics_long.csv"),
  row.names = FALSE,
  fileEncoding = "UTF-8-BOM"
)
write.csv(
  cbind(PROBE_ID = rownames(feature_info), feature_info),
  file.path(output_dir, "feature_annotation.csv"),
  row.names = FALSE,
  fileEncoding = "UTF-8-BOM"
)
형식장점용도
RDSExpressionSet의 class와 내부 구조를 그대로 보존다음 R 분석에서 readRDS()로 재사용
CSVExcel과 다른 프로그램에서 쉽게 확인검토·공유·수동 수정
원본 metadata를 덮어쓰지 않습니다. 전체 metadata와 분석용으로 정리한 metadata를 별도 파일로 저장하면 그룹 결정 과정을 추적할 수 있습니다.

종합 실습: 나의 데이터셋을 분석용 객체로 만들기

  1. 본인이 선택한 GSE를 getGEO()로 내려받습니다.
  2. 포함된 GPL별 probe 수와 sample 수를 표로 정리하고 분석할 GPL을 선택합니다.
  3. exprs(), pData(), fData()의 차원을 기록합니다.
  4. 발현행렬–metadata와 발현행렬–annotation의 ID 및 순서가 일치하는지 검사합니다.
  5. characteristics를 긴 형식으로 분리하고 이용 가능한 임상정보 항목을 나열합니다.
  6. Control과 Disease를 구분하는 조건을 R 코드로 작성합니다.
  7. 각 그룹의 sample 수와 NA 그룹이 없는지 확인합니다.
  8. 동일 subject가 여러 GSM에 반복되는지 확인합니다.
  9. ExpressionSet, 발현행렬, 전체 metadata와 분석용 metadata를 각각 저장합니다.
