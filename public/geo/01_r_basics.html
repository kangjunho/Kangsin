1R Script와 코드 실행

Console에 코드를 직접 입력할 수 있지만 연구 분석은 다시 실행할 수 있어야 합니다. 따라서 코드를 R Script(.R)에 작성하고 저장합니다.

  1. RStudio에서 File → New File → R Script를 선택합니다.
  2. Source 창에 아래 코드를 입력합니다.
  3. 실행할 줄에 커서를 두고 Ctrl + Enter를 누릅니다.
  4. File → Save를 선택하여 01_r_basics.R로 저장합니다.
# # 뒤의 문장은 설명을 위한 주석이며 실행되지 않습니다.
1 + 1
print("Hello, GEO analysis!")
좋은 습관: Console에만 입력하지 말고 분석에 사용한 코드는 Script에 남겨야 같은 분석을 재현할 수 있습니다.

2작업 폴더 이해하기

작업 폴더(working directory)는 R이 파일을 읽고 저장할 때 기준으로 사용하는 폴더입니다. 파일명만 입력했을 때 R은 이 폴더를 기준으로 파일을 찾습니다.

현재 작업 폴더 확인

getwd()

작업 폴더 변경

Windows에서는 다음 두 표현을 모두 사용할 수 있습니다.

# 방법 1: 슬래시(/) 사용 — 가장 간단하고 권장
setwd("C:/Users/admin/OneDrive/Desktop/Bioinformatics/GEO_microarray/")

# 방법 2: 역슬래시를 두 번(\\) 사용
setwd("C:\\Users\\admin\\OneDrive\\Desktop\\Bioinformatics\\GEO_microarray/")

# 변경된 작업 폴더 확인
getwd()

왜 \를 두 번 써야 할까요?

Windows 탐색기에서는 폴더를 C:\Users\admin처럼 표시합니다. 하지만 R 문자열에서 단일 역슬래시(\)는 줄바꿈(\n), 탭(\t)과 같은 특수문자를 시작하는 escape 문자입니다.

따라서 R 경로에서는 /를 사용하거나, 실제 역슬래시 하나를 표현하기 위해 \\를 사용합니다. 즉 /와 \\가 문자로 같은 것은 아니지만, Windows의 R 파일 경로에서는 두 방식이 같은 위치를 가리킬 수 있습니다.

잘못된 예

# 단일 역슬래시는 오류 또는 의도하지 않은 escape 처리를 유발
setwd("C:\Users\admin\OneDrive\Desktop\Bioinformatics\GEO_microarray")
OneDrive 경로 주의: 일반적인 실습 파일에는 사용할 수 있지만, 동기화 중 파일 잠금이나 긴 경로 문제가 생길 수 있습니다. 대용량 GEO 파일이나 반복 분석은 C:/Bioinformatics/GEO_microarray/처럼 OneDrive 밖의 짧은 영문 경로가 더 안정적입니다. 마지막의 /는 있어도 되고 없어도 됩니다.

폴더와 파일 확인

# 현재 폴더의 파일과 하위 폴더 확인
list.files()

# 폴더가 없다면 생성
dir.create("data_raw", showWarnings = FALSE)
dir.create("results", showWarnings = FALSE)

# 특정 파일이 존재하는지 확인
file.exists("data_raw")
이 과정의 권장 방식: setwd()는 개념을 이해하기 위해 배우지만, 실제 분석에서는 Step 00에서 만든 RStudio Project를 열어 작업 폴더를 자동 설정합니다. 개인 컴퓨터의 절대경로를 Script에 반복해서 적으면 다른 컴퓨터에서 코드가 작동하지 않을 수 있습니다.

3객체와 자료형

R에서는 숫자, 문자, 표와 분석 결과를 객체(object)에 저장합니다. 할당 연산자 <-는 오른쪽 값을 왼쪽 이름에 저장한다는 뜻입니다.

# 객체 만들기
sample_number <- 20
disease_name <- "Major depressive disorder"
is_case <- TRUE

sample_number
disease_name
is_case

class(sample_number)
class(disease_name)
class(is_case)
자료형의미예시
numeric숫자20, 3.14
character문자"Control"

factor범주형 그룹Control, Disease
객체 이름은 영문으로 시작하고 의미가 드러나게 작성합니다. 예: expr_matrix, sample_info, deg_result

4벡터와 인덱싱

벡터는 같은 종류의 여러 값을 하나로 묶는 R의 기본 자료구조입니다. c() 함수로 만듭니다.

# 숫자와 문자 벡터
age <- c(34, 42, 51, 29, 47)
group <- c("Control", "Control", "Disease", "Disease", "Disease")

# 길이와 값 확인
length(age)
age

# 위치로 선택: R의 위치 번호는 1부터 시작
age[1]
age[2:4]

# 조건으로 선택
age[age >= 40]
group[group == "Disease"]
중요: 비교할 때 ==를 사용합니다. <-는 값을 저장하고, ==는 두 값이 같은지 비교합니다.

5데이터프레임

데이터프레임(data frame)은 행과 열로 구성된 표 형태의 객체입니다. 각 행은 하나의 관측치 또는 샘플이고, 각 열은 해당 샘플에서 측정한 변수입니다. GEO 분석에서는 sample metadata와 DEG 결과가 주로 데이터프레임으로 저장됩니다.

구성의미sample_info 예
행(row)하나의 관측치 또는 샘플GSM001 한 개
열(column)샘플에서 측정한 변수group, sex, age
셀(cell)특정 샘플의 특정 변수 값GSM001의 age = 41

예제 데이터프레임 만들기

sample_info <- data.frame(
  sample_id = paste0("GSM00", 1:8),
  group = c("Control", "Control", "Control", "Control",
            "Disease", "Disease", "Disease", "Disease"),
  sex = c("F", "M", "F", "M", "F", "M", "F", "M"),
  age = c(41, 38, 45, 52, 49, 55, 61, 44),
  marker_expression = c(5.2, 4.8, 5.5, 5.0, 7.1, 6.8, 7.5, 6.2)
)

sample_info

구조와 크기 확인

class(sample_info)       # 객체 종류
str(sample_info)         # 각 열의 자료형과 구조
dim(sample_info)         # 행 수와 열 수
nrow(sample_info)        # 행 수: 샘플 수
ncol(sample_info)        # 열 수: 변수 수
names(sample_info)       # 열 이름
head(sample_info, 3)     # 처음 3행

행과 열 선택

# 기본 형식: 데이터프레임[행, 열]
sample_info[1, ]                         # 첫 번째 행
sample_info[, "age"]                    # age 열
sample_info$age                          # $로 age 열 선택
sample_info[1:3, c("sample_id", "age")] # 1~3행의 두 열

조건에 맞는 행 추출

# Disease 샘플
sample_info[sample_info$group == "Disease", ]

# 50세 이상 샘플
sample_info[sample_info$age >= 50, ]

# Disease이면서 여성인 샘플: &는 AND
sample_info[
  sample_info$group == "Disease" & sample_info$sex == "F",
]

# Control이거나 60세 이상인 샘플: |는 OR
sample_info[
  sample_info$group == "Control" | sample_info$age >= 60,
]

새로운 열 추가와 요약 계산

# 연령 범주 추가
sample_info$age_group <- ifelse(
  sample_info$age >= 50,
  "50_or_older",
  "under_50"
)

# 그룹별 샘플 수
table(sample_info$group)

# Disease 그룹의 평균 연령
mean(sample_info$age[sample_info$group == "Disease"])

# 그룹별 marker 평균
aggregate(
  marker_expression ~ group,
  data = sample_info,
  FUN = mean
)
sample_info[행, 열] 순서입니다. 쉼표 앞은 행 조건, 뒤는 가져올 열을 의미합니다. 조건식은 각 행마다 TRUE 또는 FALSE를 만들고, R은 TRUE인 행만 남깁니다.

6자주 사용하는 기본 내장함수

함수는 입력값을 받아 정해진 작업을 수행합니다. 기본 구조는 함수이름(입력값, 옵션)입니다.

함수용도예시
c()값을 벡터로 묶기c(1, 2, 3)
length()값의 개수length(age)
dim()행과 열 개수dim(sample_info)

str()객체의 내부 구조str(sample_info)
class()객체 종류class(age)
summary()기초 요약 통계summary(age)
mean() / median()평균/중앙값mean(age)
min() / max()최솟값/최댓값max(age)
sum()합계 또는 TRUE 개수sum(age >= 40)
table()범주별 빈도table(group)
unique()중복 없는 값unique(group)
is.na()결측값 확인sum(is.na(age))
sort()값 정렬sort(age)
names()열 또는 요소 이름names(sample_info)

결측값이 있을 때

age_with_na <- c(34, 42, NA, 29, 47)

mean(age_with_na)                 # 결과: NA
mean(age_with_na, na.rm = TRUE)   # NA를 제외하고 계산

7R package란?

R package는 특정 분석을 위해 만들어진 함수, 설명서, 예제 데이터 등을 하나로 묶은 확장 도구입니다.

기본 R은 계산기와 기본 공구가 들어 있는 도구상자입니다.

R package는 GEO 다운로드, DEG 분석, 그래프 작성 등 특정 작업에 필요한 전문 공구를 추가하는 것입니다.

패키지GEO 과정에서의 역할저장소
GEOqueryGEO 데이터와 임상정보 다운로드Bioconductor
limmaMicroarray DEG 분석Bioconductor
Biobase발현 데이터 객체 관리Bioconductor
AnnotationDbiProbe와 유전자 정보 연결Bioconductor
ggplot2분석 결과 시각화CRAN
pheatmap발현 heatmap 작성CRAN

8CRAN과 Bioconductor

구분CRANBioconductor
주요 목적통계, 데이터 처리, 시각화 등 범용 R 패키지유전체와 생물정보학 분석 패키지
설치 함수install.packages()BiocManager::install()
예시ggplot2, pheatmap, dplyrGEOquery, limma, Biobase
홈페이지CRANBioconductor

Bioconductor는 생물학적 데이터를 정확하고 재현 가능하게 분석하기 위한 오픈소스 프로젝트입니다. R과 Bioconductor의 버전 호환성을 관리하기 위해 공식적으로 BiocManager::install() 사용을 권장합니다.

버전 번호를 코드에 고정하지 않습니다. BiocManager::install()이 현재 설치된 R에 맞는 정식 Bioconductor 버전을 자동으로 선택하게 합니다.

9패키지 설치와 불러오기

CRAN 패키지 설치

# 설치는 컴퓨터에서 처음 한 번만 수행
install.packages(c("ggplot2", "pheatmap"))

Bioconductor 패키지 설치

# BiocManager가 없을 때만 설치
if (!requireNamespace("BiocManager", quietly = TRUE)) {
  install.packages("BiocManager")
}

# GEO microarray 실습 패키지 설치
BiocManager::install(c(
  "GEOquery",
  "limma",
  "Biobase",
  "AnnotationDbi"
))

설치한 패키지 불러오기

# R을 새로 시작할 때마다 사용 전에 실행
library(GEOquery)
library(limma)
library(ggplot2)
명령실행 시점의미
install.packages()패키지가 없을 때 한 번CRAN 패키지를 컴퓨터에 설치
BiocManager::install()패키지가 없을 때 한 번Bioconductor 패키지를 설치
library()R을 시작할 때마다설치된 패키지를 현재 분석에서 사용
흔한 실수: 패키지를 설치했더라도 library()로 불러오지 않으면 해당 함수가 검색되지 않을 수 있습니다.

패키지 이름 없이 함수 지정하기

# package::function 형식
GEOquery::getGEO
limma::lmFit

::는 여러 패키지에 같은 이름의 함수가 있을 때 어떤 패키지의 함수인지 명확하게 지정합니다.

전체 샘플 수, 변수 수, 열 이름과 각 열의 자료형을 확인하세요.

문제 2. 단일 조건 추출

Disease 그룹이면서 여성인 샘플의 sample_id, age, marker_expression만 추출하세요.

문제 4. 연령 조건

50세 이상인 샘플은 몇 명인지 계산하고, 해당 샘플 전체 정보를 출력하세요.

문제 5. 그룹별 계산

Disease 그룹의 평균 연령과 Control·Disease별 평균 marker expression을 계산하세요.
