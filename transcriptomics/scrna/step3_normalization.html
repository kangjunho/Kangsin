Normalization and Feature Selection

Step 03 / 08

QC 이후 데이터는 세포 간 library size 차이를 보정하고, 분석에 중요한 유전자(variable genes)를 선택하는 과정을 거칩니다.

A) 왜 Normalization이 필요한가?

  • 각 세포는 sequencing depth가 다름
  • 같은 유전자라도 count 값이 직접 비교 불가능
  • 따라서 scale을 맞춰주는 과정이 필요
핵심 개념
  • Normalization = library size correction
  • 비교 가능한 값으로 변환

B) LogNormalize 방법


obj <- NormalizeData(
  obj,
  normalization.method = "LogNormalize",
  scale.factor = 10000
)
  • 각 cell을 total count 기준으로 scaling
  • log transformation 적용

C) SCTransform (추천 방법)


obj <- SCTransform(
  obj,
  vars.to.regress = "percent.mt",
  verbose = FALSE
)
  • variance stabilization 수행
  • technical noise 제거
  • 최근 분석에서는 SCTransform 권장

D) Variable Feature Selection


obj <- FindVariableFeatures(
  obj,
  selection.method = "vst",
  nfeatures = 2000
)
  • 모든 유전자가 아니라 중요한 유전자만 선택
  • noise 줄이고 signal 강화

E) Scaling


obj <- ScaleData(obj)
  • 평균 0, 분산 1로 정규화
  • PCA 전 필수 단계

F) 결과 확인


VariableFeaturePlot(obj)

top10 <- head(VariableFeatures(obj), 10)
LabelPoints(plot = VariableFeaturePlot(obj), points = top10)

G) 전체 코드 정리


# Normalize
obj <- NormalizeData(obj)

# Variable genes
obj <- FindVariableFeatures(obj, nfeatures = 2000)

# Scaling
obj <- ScaleData(obj)

# Check
VariableFeaturePlot(obj)
요약
  • Normalization은 세포 간 비교를 가능하게 함
  • Variable gene selection은 signal을 강화
  • Scaling은 PCA 준비 단계
