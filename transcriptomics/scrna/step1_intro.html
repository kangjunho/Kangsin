Step 01 / 08

본 실습에서는 mouse colorectal cancer single-cell RNA-seq 데이터를 사용하여 scRNA-seq의 개념부터 실제 분석 환경까지 이해하고, 이후 단계에서 분석을 수행할 준비를 합니다.

A) Transcriptomics 기술의 발전

유전자 발현 분석 기술은 지속적으로 발전해 왔으며, 각 단계는 이전 기술의 한계를 보완하는 방향으로 진화해 왔습니다.

초기의 Microarray 기술은 수천 개 유전자의 발현을 동시에 측정할 수 있었지만, 미리 정의된 probe에 의존한다는 제한이 있었습니다. 이후 등장한 Bulk RNA-seq은 sequencing 기반으로 보다 정확한 발현 측정이 가능해졌지만, 여전히 하나의 샘플을 평균값으로 분석한다는 한계를 가지고 있습니다.

이러한 한계를 해결하기 위해 등장한 Single-cell RNA-seq은 개별 세포 단위에서 유전자 발현을 측정하여 세포 간 이질성을 직접 분석할 수 있게 합니다.

최근에는 Spatial transcriptomics 기술이 등장하여, 유전자 발현뿐 아니라 세포의 위치 정보까지 함께 분석할 수 있는 단계로 발전하고 있습니다.

기술 흐름
Microarray → Bulk RNA-seq → Single-cell RNA-seq → Spatial transcriptomics

B) Bulk vs Single-cell RNA-seq

Bulk RNA-seq는 조직 전체에서 RNA를 추출하여 분석하기 때문에, 측정된 발현값은 여러 세포의 평균값을 의미합니다. 따라서 서로 다른 세포 유형이 섞여 있을 경우 개별 세포의 특성을 구분하기 어렵습니다.

반면, Single-cell RNA-seq는 각 세포를 개별적으로 분석하여, 세포 간 차이와 다양한 세포 상태를 직접적으로 확인할 수 있습니다.

C) scRNA-seq 핵심 개념

scRNA-seq 데이터는 gene × cell 형태의 matrix로 구성되며, 각 세포는 고유한 barcode를 통해 구분됩니다. 또한 UMI는 PCR 중복을 제거하여 실제 분자 수를 보다 정확하게 반영합니다.

이 데이터의 특징은 많은 값이 0으로 나타나는 sparsity이며, 이는 기술적 요인과 biological 특성이 함께 반영된 결과입니다.

D) 분석 환경: RStudio Server

본 실습은 로컬 PC가 아닌 리눅스 서버 기반 RStudio Server에서 수행됩니다. 

http://server_ip:8787

웹 브라우저에서 접속 후 ID / Password 입력하여 로그인합니다.

E) NAS 데이터 접근 (중요)

실습 데이터는 NAS(Network Attached Storage)에 저장되어 있으며, 분석 서버에 원격 마운트(remote mount) 되어 있어야 접근 가능합니다.

핵심 개념
사용자는 웹브라우저로 서버에 접속하지만,
실제 데이터는 NAS에서 불러옵니다. 
~/God_nas/Mouse_CRC/

F) 데이터 구조


Mouse_CRC/
├── Raw_data/
└── SevenBridges/
  • Raw_data → 원시 데이터
  • SevenBridges → 분석 결과 및 Seurat object

G) 분석용 데이터 (.rds)


~/God_nas/Mouse_CRC/SevenBridges/ND1/
└── DA250003-ND1_Seurat.rds

실제 분석은 이 Seurat object를 기반으로 수행됩니다.

H) 데이터 불러오기


library(Seurat)
